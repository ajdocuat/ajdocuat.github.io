    ajBigNumPrecision function

    Description

    Returns a BigNumber whose value is the value of this BigNumber rounded to a precision defined in trailing_zeros.

    Syntax

    ajBigNumPrecision(number, [digits], [rounding_mode], [run_condition], [run_by_function_point_only])

    Argument NameArgument TypeDescription
    number
    (required)    		StringIt is to specify the number, string or BigNumber.
    digits
    (optional)    		VariantIf it is a number, returns a BigNumber whose value is the value of this BigNumber rounded to a precision of significant digits using rounding mode.
    If it is omitted or is null, the return value is the number of significant digits of the value of this BigNumber, or null if the value of this BigNumber is ±Infinity or NaN.
    If it is true then any trailing zeros of the integer part of a number are counted as significant digits, otherwise they are not.
    round_mode
    (optional)    		IntegerIt is to define the rounding mode. Here is the supported rounding mode:
    0 - ROUND_UP,
    1 - ROUND_DOWN,
    2 - ROUND_CEIL,
    3 - ROUND_FLOOR,
    4 - ROUND_HALF_UP,
    5 - ROUND_HALF_DOWN,
    6 - ROUND_HALF_EVEN,
    7 - ROUND_HALF_CEIL,
    8 - ROUND_HALF_FLOOR.

    If you do not specify anything, the default value will always be 4 - ROUND_HALF_UP.

    run_condition
    (optional)    		BooleanThe function will run when the value is TRUE. Otherwise will not run.

    If you do not specify anything, the default value will always be TRUE.

    run_by_function_point_only
    (optional)    		BooleanIf it equals FALSE, the function can be executed through ‘Excel Calculation’ (can be either Automatic or Manual, Calculate Now or Calculate Sheet) or Preview Function Point.

    If it equals TRUE, the function can be executed with Preview Function Point (AlchemyJ toolbar \ Preview Function Point) only.

    If you do not specify anything, the default value will always be TRUE.

    The function will return:

    1) Content type: BigNumber

    2) Method: Within a cell

    Error Scenarios

    System error will be raised for the below scenarios.

    ParameterError Message
    numberNumber should be numeric.
    digitsDigits must be in boolean or integer data type.
    rounding_modeRound mode must be between 0 and 8.
